# 深分页为什么慢, 怎么优化

## 题目

> 深分页为什么慢？一般怎么优化？

## 参考答案

因为 `limit offset, size` 在 offset 很大时，要先扫描并跳过前面大量记录。

常见优化：

- 用游标翻页，比如基于 id
- 先查主键，再回表
- 限制最大翻页深度

所以“页码越往后越慢”本质上是扫描成本变大。
## 回答展开

- SQL 和索引题的关键是解释为什么快、为什么慢，而不是只背优化口诀。
- 回答时最好结合执行计划、扫描行数、回表次数和写入成本。
- 真正有经验的回答通常会强调以执行计划和真实数据分布为准。

## 实际场景

比如后台订单列表查询慢，通常要先看执行计划，再判断是不是没走索引、回表太多或深分页导致扫描成本过高。
