# 如何设计一个高并发签到/点赞系统

## 题目

> 如何设计一个高并发签到 / 点赞系统？

## 参考答案

典型做法：

- Redis 做实时计数
- Bitmap / Set 记录是否操作过
- MQ 异步落库
- 定时汇总统计

重点在于防重复、热点打散和最终对账。
## 回答展开

- 系统设计题建议先讲目标和约束，再给核心链路，最后补高可用与降级。
- 回答时尽量体现读写路径、热点、容量、故障处理和扩展性。
- 如果能说明为什么这样拆层、为什么这样选型，会更像真实设计。

## 实际场景

这些都是典型系统设计题。比如秒杀要解决削峰和防超卖，排行榜要解决实时排序，通知系统要解决模板、多渠道和失败重试。
