## 📖 参考答案

### 一、JVM内存结构（JDK8+）

```
┌─────────────────────────────────────────────────┐
│                    JVM 内存                      │
├──────────────────┬──────────────────────────────┤
│   线程共享区域     │        线程私有区域           │
├──────────────────┼──────────────────────────────┤
│  ┌────────────┐  │  ┌─────────────────────────┐ │
│  │    堆      │  │  │   虚拟机栈（VM Stack）   │ │
│  │ (Heap)     │  │  │   - 栈帧（局部变量表、   │ │
│  │ - 新生代   │  │  │     操作数栈、动态链接） │ │
│  │ - 老年代   │  │  └─────────────────────────┘ │
│  └────────────┘  │  ┌─────────────────────────┐ │
│  ┌────────────┐  │  │   本地方法栈            │ │
│  │  元空间    │  │  │   (Native Method Stack) │ │
│  │(Metaspace) │  │  └─────────────────────────┘ │
│  │ - 类信息   │  │  ┌─────────────────────────┐ │
│  │ - 常量池   │  │  │   程序计数器（PC）       │ │
│  └────────────┘  │  └─────────────────────────┘ │
└──────────────────┴──────────────────────────────┘
         ↓
   堆外内存：直接内存（Direct Memory）
```

### 二、JVM调优经验

```bash
# 基础参数
-Xms4g -Xmx4g          # 堆大小，建议设成一样避免动态扩容
-Xmn2g                  # 新生代大小
-XX:MetaspaceSize=256m  # 元空间初始值
-XX:MaxMetaspaceSize=512m

# GC选择（JDK8常用）
-XX:+UseG1GC            # G1收集器
-XX:MaxGCPauseMillis=200 # 目标停顿时间

# 监控参数
-XX:+PrintGCDetails
-XX:+PrintGCDateStamps
-Xloggc:/path/to/gc.log
```

### 三、OOM案例（示范回答）

> "之前遇到一个OOM问题：某个定时任务批量导出Excel，导致Heap OOM。
> 
> **排查过程**：
> 1. 启动参数添加 `-XX:+HeapDumpOnOutOfMemoryError -XX:HeapDumpPath=/tmp/dump`
> 2. 复现问题后，用MAT打开dump文件
> 3. 通过 Dominator Tree 发现一个 `ArrayList` 占用了1.2G
> 4. 追溯到代码：一次性加载了50万条数据到内存
> 
> **解决方案**：
> - 改为分页查询 + 流式写入Excel（使用EasyExcel的SXSSFWorkbook）
> - 限制单次导出上限1万条"
