## 📖 参考答案

### 一、核心目标

- 互斥
- 避免死锁
- 可重试
- 最好支持续期和可观测

### 二、Redis 分布式锁常见实现

```redis
SET lock_key unique_value NX PX 30000
```

关键点：

- `NX`：只在 key 不存在时加锁
- `PX`：设置过期时间，避免死锁
- `unique_value`：解锁时校验，避免误删别人的锁

### 三、安全释放锁

需要用 Lua 保证校验 value 和删除 key 的原子性。

### 四、常见问题

- 业务执行时间超过锁过期时间
- Redis 主从切换导致锁丢失
- 锁竞争激烈时吞吐下降
- 抢到锁的客户端宕机
