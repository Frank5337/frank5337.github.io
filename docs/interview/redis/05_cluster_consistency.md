# Redis集群中, 多个节点是怎么保证数据一致性的

## 题目

> Redis 集群里多个节点如何保证数据一致性？

## 参考答案

Redis 主要依赖主从复制来同步数据，但默认是异步复制，所以它更偏最终一致性，而不是强一致。

面试里要明确两点：

- 主从切换时可能丢少量数据
- Redis 不适合承担强事务一致性场景
## 回答展开

- 这题要先把预期讲清楚：Redis 集群并不提供数据库那种强一致，它依赖主从复制，默认更偏最终一致。
- 所以回答重点不该是怎么做到完全不丢，而是复制机制是什么、主从切换时会有什么风险、业务上怎么兜底。
- 如果面试官继续追问，通常会问到异步复制、主从切换数据丢失窗口，以及为什么核心数据不能只放 Redis。

## 实际场景

比如把 Redis 作为库存或会话的高速访问层没问题，但最终订单结算、资金记账仍然要以数据库为准，因为主从切换时 Redis 可能丢掉极少量尚未同步的数据。
