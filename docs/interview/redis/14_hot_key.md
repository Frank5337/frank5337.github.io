# 热Key问题怎么发现和处理

## 题目

> 热 Key 问题怎么发现和处理？

## 参考答案

热 key 会让单个分片承受异常高流量。常见处理方案：

- 本地缓存
- 热点预热
- key 拆分
- 多副本或代理层缓存

重点是把流量从单点打散。
## 回答展开

- 缓存题建议按原理、风险、治理方案三个层次展开。
- 如果涉及集群或高可用，要明确 Redis 更偏最终一致而不是强一致。
- 如果涉及缓存问题，最好把穿透、击穿、雪崩的成因和治理手段区分开。

## 实际场景

比如商品详情缓存、用户会话缓存、配置缓存是 Redis 的典型使用场景；一旦出现缓存失效、热点集中或删除失败，就需要完整的治理策略。
